# RemnaWave Panel Installation Script

Скрипт автоматической установки и управления панелью RemnaWave для Debian/Ubuntu серверов.

## 📋 Возможности

### Основные функции:
1. **Чистая установка** — полная установка панели с нуля
2. **Миграция панели** — перенос с другого сервера через SSH (с поддержкой смены домена или только IP)
3. **Бэкап панели** — создание архива с базой данных и конфигурацией
4. **Восстановление** — восстановление из бэкапа (на том же или новом сервере)
5. **Обновление** — безопасное обновление с автоматическим бэкапом
6. **Удаление** — полная деинсталляция панели

### Поддерживаемые веб-серверы:
- **Nginx + Certbot** — классический вариант с Let's Encrypt SSL
- **Caddy** — современный веб-сервер с автоматическим SSL

wget -O install.sh https://raw.githubusercontent.com/Armruswest2024/remnainstabac/refs/heads/main/install.sh && chmod +x install.sh && ./install.sh

# Сделайте его исполняемым

### Требования к системе

- **ОС**: Debian 10+/Ubuntu 20.04+
- **RAM**: минимум 2GB (рекомендуется 4GB+)
- **Диск**: минимум 10GB свободного места
- **Права**: запуск от root (или через sudo)
- **Порты**: 80 и 443 должны быть свободны

## 📖 Использование

После запуска скрипта вы увидите интерактивное меню:

```
╔════════════════════════════════════════╗
║   RemnaWave Panel Installer v1.1       ║
╚════════════════════════════════════════╝

Главное меню:
  1) Чистая установка панели
  2) Миграция панели (с другого сервера)
  3) Создать бэкап панели
  4) Восстановить из бэкапа
  5) Обновить панель
  6) Удалить панель
  7) Показать статус
  8) Выход
```

### 1. Чистая установка

Выберите опцию `1` в главном меню.

**Процесс установки:**
1. Ввод домена (например, `panel.example.com`)
2. Ввод email для SSL уведомлений
3. Выбор веб-сервера (Nginx или Caddy)
4. Генерация учетных данных администратора
5. Установка зависимостей (PostgreSQL, веб-сервер)
6. Настройка базы данных
7. Развертывание приложения
8. Настройка SSL сертификата
9. Создание systemd сервиса

**После установки вы получите:**
- URL доступа к панели (https://ваш-домен)
- Логин и пароль администратора
- Доступ к логам и командам управления

### 2. Миграция панели

Выберите опцию `2` в главном меню.

**Поддерживаемые сценарии:**
- ✅ Перенос на новый сервер с тем же доменом
- ✅ Перенос на новый сервер со сменой домена
- ✅ Перенос только по IP (без домена)

**Что нужно указать:**
1. IP адрес старого сервера
2. SSH порт (по умолчанию 22)
3. Путь к файлу бэкапа на старом сервере
4. Опционально: новый домен и email

**Процесс миграции:**
1. Загрузка бэкапа по SSH
2. Восстановление базы данных
3. Восстановление файлов приложения
4. Обновление конфигурации (если сменился домен)
5. Перевыпуск SSL сертификата (при смене домена)
6. Запуск сервиса

### 3. Бэкап панели

Выберите опцию `3` в главном меню.

**Что включается в бэкап:**
- Дамп PostgreSQL базы данных
- Конфигурационный файл (`.env` в `/opt/remnawave/`)
- Файлы приложения (`/opt/remnawave/`)

**Автоматические бэкапы:**
- Скрипт настраивает cron job на ежедневный бэкап в 3:00
- Хранятся последние 7 бэкапов
- Расположение: `/var/backups/remnawave/`

**Ручной бэкап:**
```bash
/opt/remnawave/scripts/backup.sh
```

### 4. Восстановление из бэкапа

Выберите опцию `4` в главном меню.

**Сценарии использования:**
- Восстановление после сбоя на том же сервере
- Перенос на новый сервер вручную
- Откат к предыдущей версии

**Процесс:**
1. Выбор файла бэкапа из списка доступных
2. Остановка сервиса RemnaWave
3. Восстановление базы данных
4. Восстановление файлов
5. Обновление конфигурации
6. Запуск сервиса

### 5. Обновление панели

Выберите опцию `5` в главном меню.

**Процесс обновления:**
1. Автоматическое создание бэкапа
2. Загрузка новой версии с GitHub Releases
3. Остановка сервиса
4. Замена файлов приложения
5. Запуск миграций базы данных
6. Перезапуск сервиса
7. Проверка статуса

**Важно:** Перед обновлением рекомендуется вручную создать дополнительный бэкап.

### 6. Удаление панели

Выберите опцию `6` в главном меню.

**Что удаляется:**
- Файлы приложения (`/opt/remnawave/`)
- Конфигурационный файл (`.env` в `/opt/remnawave/`)
- База данных PostgreSQL
- Конфигурация веб-сервера
- Systemd сервис
- Cron jobs для бэкапов

**Сохраняется:**
- Логи установки (опционально)
- Бэкапы в `/var/backups/remnawave/`

## 🔧 Технические детали

### Структура установки

```
/opt/remnawave/           # Приложение и конфигурация
├── .env                  # Основной конфиг (DATABASE_URL, секреты, домен)
├── bin/                  # Исполняемые файлы
│   └── remnawave         # Главный бинарник
├── public/               # Статические файлы
├── logs/                 # Логи приложения
└── scripts/              # Скрипты утилит
    └── backup.sh         # Скрипт бэкапа

/var/backups/remnawave/   # Бэкапы
└── remnawave_backup_*.tar.gz

/var/log/
├── remnawave_install.log # Лог установки
└── remnawave_backup.log  # Лог бэкапов
```

### Конфигурационный файл

Файл `/opt/remnawave/.env` содержит:

```bash
NODE_ENV=production
PORT=3000
DATABASE_URL=postgresql://remnawave:PASSWORD@localhost:5432/remnawave
JWT_SECRET=SECRET_KEY
ADMIN_USERNAME=admin
ADMIN_PASSWORD=GENERATED_PASSWORD
DOMAIN=panel.example.com
WEB_SERVER=nginx
```

### Systemd сервис

```ini
[Unit]
Description=RemnaWave Panel
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/remnawave
EnvironmentFile=/opt/remnawave/.env
ExecStart=/opt/remnawave/bin/remnawave start
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### Полезные команды

```bash
# Статус сервиса
systemctl status remnawave

# Перезапуск
systemctl restart remnawave

# Просмотр логов
journalctl -u remnawave -f

# Логи установки
tail -f /var/log/remnawave_install.log

# Ручной бэкап
/opt/remnawave/scripts/backup.sh

# Список бэкапов
ls -lh /var/backups/remnawave/
```

## 🔐 Безопасность

### Что делает скрипт:
- ✅ Генерирует случайные пароли для БД и JWT
- ✅ Настраивает firewall (UFW) для портов 22, 80, 443
- ✅ Использует md5 аутентификацию для PostgreSQL
- ✅ Настраивает security headers в веб-сервере
- ✅ Ограничивает права доступа к конфигам (chmod 600)
- ✅ Использует HTTPS с автоматическим SSL

### Рекомендации:
1. Сохраните сгенерированные пароли в надежном месте
2. Регулярно проверяйте наличие обновлений
3. Настройте мониторинг доступности панели
4. Периодически тестируйте восстановление из бэкапа

## 🐛 Решение проблем

### Логи для диагностики

```bash
# Лог установки
cat /var/log/remnawave_install.log

# Лог сервиса
journalctl -u remnawave --since "1 hour ago"

# Лог Nginx
tail -f /var/log/nginx/error.log

# Лог PostgreSQL
tail -f /var/log/postgresql/postgresql-*.log
```

### Частые проблемы

**1. Порт 80/443 занят**
```bash
# Проверить что занимает порт
ss -tlnp | grep :80
ss -tlnp | grep :443

# Остановить конфликтующий сервис
systemctl stop apache2  # или другой веб-сервер
```

**2. Не получается SSL сертификат**
```bash
# Проверить DNS записи
dig panel.example.com

# Проверить доступность порта 80
curl -I http://panel.example.com

# Получить сертификат вручную
certbot certonly --webroot -w /var/www/certbot -d panel.example.com
```

**3. Сервис не запускается**
```bash
# Проверить статус
systemctl status remnawave

# Проверить логи
journalctl -u remnawave -n 50

# Проверить базу данных
pg_isready -h localhost
```

**4. Проблемы с миграцией**
```bash
# Проверить SSH подключение
ssh -p PORT root@SOURCE_IP

# Проверить наличие бэкапа
ls -lh /path/to/backup.tar.gz

# Попробовать скачать вручную
scp -p PORT root@SOURCE_IP:/path/to/backup.tar.gz /tmp/
```
